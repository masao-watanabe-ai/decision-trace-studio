# Decision Trace Studio — デモ台本（日本語）

---

## 1分版デモ台本

---

「このツールは、コールセンターの対応フローを設計して、シミュレーションして、改善まで一気通貫でできる Studio です。」

「まず Design タブを見てください。VIP のお客様は即座に優先対応、法的リスクがある場合は境界処理、クレームは人間に回す、という判断フローがノードとして並んでいます。」

「次に Simulate タブ。シナリオを自動生成して全件実行すると、各ノードの通過経路とアクションが記録されます。」

「そしてこれが一番大事な点ですが、判断を記録するだけでなく、Improve タブで改善提案を自動生成できます。提案を承認すると、フローにそのまま反映されます。」

「AIが答えを出すツールではありません。人間が決めた判断基準を設計して、実行して、改善し続けるための Studio です。」

## 3分版デモ台本

---

### オープニング（約20秒）

「今日は Decision Trace Studio をご紹介します。一言で言うと、意思決定を設計・実行・記録・改善するためのツールです。」

「よく似たツールとして AI の出力を活用するものがありますが、このツールの立場は少し違います。AI はシグナルを出す存在、つまり「この可能性があります」と教えてくれる存在です。一方でビジネスの現場では、そのシグナルを受けて**最終的に何をするか**、つまり Decision を下す必要があります。このツールは、その Decision の部分を設計・管理するためのものです。」

---

### Design タブ（約30秒）

「まず Design タブを見てください。これがコールセンター対応フローです。」

「ノードは4種類あります。通常の判断をおこなう Decision ノード、リスクや境界条件を捉える Boundary ノード、必ず人間が判断する Human Gate ノード、そしてどの条件にも当てはまらないときの Fallback ノードです。」

「たとえばここの Refund+Legal Risk ノードは、払い戻し請求が一定金額を超えていて、かつ法的な言及がある場合に境界として機能します。このノードに引っかかったケースは、自動対応ではなくエスカレーションのアクションが実行されます。」

「ノードの追加・削除・接続はドラッグ操作でできます。変更を保存するとバージョンが上がります。」

---

### Simulate タブ（約30秒）

「次に Simulate タブです。シナリオ生成ボタンを押すと、このフローに対してテストケースが自動で作られます。」

「全件実行すると、各シナリオがフローを走行して、どのノードを通過してどのアクションが選ばれたかがすべて記録されます。」

「各行の Trace リンクを開くと、そのシナリオがノードごとにどう評価されたかのステップを詳細に追えます。「なぜこの判断になったのか」が、後から確認できる状態になっています。」

---

### Compare タブ（約30秒）

「ここが Compare タブです。フローを変更する前と後のシミュレーション結果を並べて比較できます。」

「たとえばノードを一つ追加したときに、どのシナリオのアクションが変わったか、逆にどのシナリオは影響を受けなかったか、が赤と緑のハイライトで一目でわかります。」

「改善が意図した効果をもたらしているか、予期しない副作用がないかを、定量的に確認してから本番に反映できます。」

---

### Improve タブ（約30秒）

「Improve タブでは、シミュレーション結果を分析して改善提案を自動生成します。」

「たとえば「このシナリオ群で Fallback に落ちているケースが多い。優先度の高い Boundary ノードを追加することで捌けるはず」というような提案が出てきます。」

「提案の内容を確認して、承認ボタンを押すと Flow に即反映されます。その後 Simulate を再実行して Compare で差分を見れば、改善の効果が数値で確認できます。」

---

### クロージング（約20秒）

「設計して、実行して、比較して、改善する。このサイクルを一つの Studio の中で回せるのが、Decision Trace Studio の価値です。」

「AI に判断を任せるのではなく、人間の判断基準をフローとして設計して、その品質を継続的に高めていく。そういう使い方を想定しています。」

「ご質問があればぜひ。ありがとうございました。」

## 想定画面操作メモ

| タイミング | 操作 |
|---|---|
| オープニング | StudioPage を開いた状態（Design タブ表示中） |
| Design 説明 | ノードをクリックして NodeDetailPanel に条件・アクションを表示する |
| Design 説明 | Boundary ノード（Refund+Legal Risk）を選択してハイライト |
| Simulate | シナリオ生成 → 全件実行を実演 |
| Trace | 任意のシナリオ行の Trace リンクをクリックして詳細ステップを見せる |
| Compare | Before / After のカラムが並んだ状態を表示（事前にシミュレーション2回実行済み） |
| Improve | 提案一覧を表示 → 承認ボタンをクリック → Flow に反映されることを Design タブで確認 |
| クロージング | Guided Demo の「完了」ボタンを押してデモを終了 |

**事前準備チェックリスト**

- シミュレーションを1回実行済みの状態にしておく（Compare 用の Before データ）
- Improve の提案が少なくとも1件生成済みの状態にしておく
- ブラウザのズームを 90% 程度に下げて全体が見えるようにする
- Guided Demo モードをオフにした状態でデモを開始し、必要なタイミングで `▶ Demo` ボタンを押す

## 一言説明（短い製品説明）

**Decision Trace Studio** は、コールセンターや業務オペレーションにおける意思決定フローを設計・実行・比較・改善するための Studio です。AI の出力（シグナル）を受けて**最終的に何をするか（Decision）**を人間が設計し、そのフローをシミュレーションで検証して、継続的に品質を高めていくことができます。
